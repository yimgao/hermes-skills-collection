# YouTube Script Writer

> Turn topics, articles, products, or ideas into retention-engineered YouTube scripts. Hook in 8 seconds, pattern interrupts every 60–90s, B-roll cues for editors, clickable title + thumbnail brief, end-screen CTA — all produced in one structured draft.

YouTube is **not** a blog read aloud, **not** a podcast transcribed, **not** a sales page with a face. It rewards:

- **First 8 seconds** — viewers decide to stay or leave before any value lands
- **Open loop** — promise a payoff later so viewers don't click away mid-video
- **Pattern interrupts every 60–90 seconds** — camera change, B-roll, zoom, meme, sound effect, or new visual
- **Specificity over generality** — "I made $4,217 in 14 days" beats "I made money"
- **Payoff density** — every 2–3 minutes delivers a tangible insight, not filler
- **End-screen / card strategy** — the last 20 seconds dictate what viewers watch next

This skill enforces all of the above with measurable rules.

## Core Workflow

### Step 1: Identify the Video Format

Choose ONE structure. Each format has a specific rhythm and section template.

| Format | When to Use | Length | Section Template |
|--------|-------------|--------|------------------|
| **Tutorial / How-To** | Teaching a process, tool, or skill | 8–15 min | Hook → Promise → Prereqs → Step 1 → Step 2 → ... → Common mistakes → Recap → CTA |
| **Listicle / Top-N** | "5 plugins", "7 mistakes", curated picks | 10–20 min | Hook → Why this matters → Item 1 (full demo) → ... → Item N → Bonus → CTA |
| **Narrative / Story** | Personal story, company origin, case study | 12–25 min | Cold open (climax frame) → Rewind to start → Build-up → Tension → Resolution → Lesson → CTA |
| **Day-in-the-Life / Vlog** | Founder vlog, routine, behind-the-scenes | 10–20 min | Cold open (best moment) → Morning → Work block 1 → ... → Reflection → CTA |
| **Review / Comparison** | Product review, A vs B, hands-on | 12–20 min | Hook (verdict tease) → Context → Specs → Test 1 → Test 2 → Pros/cons → Final verdict → CTA |

Ask the user (or infer): *"What should the viewer be able to DO, FEEL, or DECIDE after watching this?"* — this decides the format and length.

### Step 2: Craft the Hook (Most Important Step)

The first **8 seconds** decide if the video gets watched past 30s. YouTube's algorithm heavily weights this.

**6 proven hook patterns:**

1. **The Result Hook** — *"This script generated $47,000 in 30 days. Let me show you exactly how."*
2. **The Question Hook** — *"Why does every React app eventually become slow?"*
3. **The Contrarian Hook** — *"Stop using React. Here's what I use instead."*
4. **The Mistake Hook** — *"You're using ChatGPT wrong. Here's the mistake almost everyone makes."*
5. **The Visual Hook** — *"[On screen: dramatic result] In the next 10 minutes I'll show you how I got here."*
6. **The Bold Promise Hook** — *"By the end of this video, you'll have a deployable side project. Let's go."*

**Hook checklist:**
- [ ] Specific number or concrete outcome (not "a lot of money", but "$4,217")
- [ ] No "Hey guys, welcome back to my channel" — never open with channel branding
- [ ] Promise payoff that's relevant to viewer's life, not to the creator's
- [ ] Under 25 words so it lands before the 8-second mark

### Step 3: Map the Retention Architecture

Before writing sentences, sketch the **beat sheet**. Place these markers:

```
[00:00] HOOK — 8s (pattern above)
[00:08] REFRAME — what's coming, why it matters
[00:30] OPEN LOOP #1 — promise of later payoff
[02:00] SECTION 1 begins — pattern interrupt (camera change / B-roll)
[04:30] PAYOFF #1 — first tangible insight
[05:00] OPEN LOOP #2 — set up next section
[07:00] SECTION 2 begins — pattern interrupt
[09:30] PAYOFF #2 — major insight
[10:00] MID-ROLL CTA — soft subscribe nudge tied to upcoming value
[12:00] SECTION 3 begins — pattern interrupt
[14:30] FINAL PAYOFF + RECAP
[15:00] END SCREEN — next video suggestion + subscribe
```

**Pattern interrupt toolbox** (rotate every 60–90s):
- Camera angle change (cut to B-roll, screen recording, or overhead)
- Zoom-in on a key prop or line of code
- Sound effect / music shift
- Memes / pop-culture reference
- Cutaway to a visual proof (screenshot, demo, result)
- On-screen text or graphic that reinforces the spoken line

### Step 4: Write the Script with Visual Markers

For each section, alternate between **spoken lines** and **visual cues**. Editors need markers; vague prose is unusable.

```
[00:00 — HOOK]

[VISUAL: Cut to me at desk, intense eye contact, no intro music]
SPOKEN: "This Python script made me $47,000 in 30 days. Today I'll show you every single line, and you'll have it running by the end of this video."

[VISUAL: B-roll of the script running, money counter on screen]
SPOKEN: "But first — three mistakes that will silently kill your results if you skip them."

[00:08 — REFRAME + OPEN LOOP #1]

[VISUAL: Animated title card with the 3 mistakes]
SPOKEN: "We're going to cover the setup, the optimization, and the part most people miss entirely — the part that 10x'd my results."

[VISUAL: Zoom-in on the script file in my editor]
SPOKEN: "Stick around, because mistake number 3 alone saved me from a $4,000 chargeback disaster last month."

[02:00 — SECTION 1: Setup]

[VISUAL: Screen recording — terminal window, fresh, empty]
SPOKEN: "Step one: create a virtual environment. Here's why this matters before we touch any code."

[B-ROLL: zoom on pip install output scrolling]
SPOKEN: "Notice we're pinning the exact versions. This isn't paranoia — it's how you avoid the most common debugging nightmare in Python."

[PATTERN INTERRUPT: cut to whiteboard diagram of package dependency tree]
SPOKEN: "Let me show you what happens when you don't do this..."

[04:30 — PAYOFF #1]
...
```

**Script rules:**
- Spoken lines average 12–18 words per sentence (breath control)
- Every section ends with a **bridge line** that creates momentum: "But here's where it gets interesting..." / "Now watch what happens when..." / "This next part is the real reason I made this video."
- Mark every visual cue with `[VISUAL: ...]`, `[B-ROLL: ...]`, `[GRAPHIC: ...]`, or `[ZOOM: ...]`
- Mark every CTA with `[CTA: ...]` so editors know to add subscribe animation

### Step 5: Produce the SEO Metadata Bundle

A great script with a bad title dies. Always produce a metadata bundle.

**Title (primary):**
- ≤60 characters (YouTube truncates at ~60 on mobile)
- Front-load the hook word ("Stop using...", "$47K in 30 days", "The mistake...")
- Create curiosity gap without clickbait that the video doesn't pay off

**Alternate titles (2–3 backups for A/B testing):**
- Same promise, different angle (result-focused, audience-focused, format-focused)

**Thumbnail brief:**
```
SUBJECT: [Who/what is on screen — be specific]
  e.g., "Me holding the broken hard drive, shocked face"

EMOTION: [The dominant emotion viewers should feel]
  e.g., "Shock + curiosity"

VISUAL HOOK: [Background / prop / color / contrast]
  e.g., "Bright red background, hard drive in foreground, clean"

3-WORD TEXT OVERLAY: [The exact text, max 3 words]
  e.g., "IT DIED"
```

**Description (with chapter timestamps):**
```
[00:00] The hook
[02:00] Mistake #1: ...
[05:30] Mistake #2: ...
[09:00] Mistake #3: ...
[12:00] The script walkthrough
[15:00] Results + next steps

Free download: [link]
My tool: [link]
Follow me on Twitter: [link]
```

### Step 6: Retention Prediction Score

Before delivering, score the script on 6 signals. Aim for ≥4/6.

| Signal | Question | Pass Criteria |
|--------|----------|---------------|
| **Hook strength** | Does the first 8s have a specific number or concrete outcome? | Yes / No |
| **Open loop count** | Are ≥2 unresolved promises made before payoff? | Count loops |
| **Pattern interrupt cadence** | Is there a visual change every 60–90s? | Count cues |
| **Payoff density** | Does a tangible insight land every 2–3 minutes? | Count payoffs |
| **Energy curve** | Does intensity rise toward climax, not sag in the middle? | Map energy 1–10 across script |
| **CTA relevance** | Does the end-screen CTA tie to a video the viewer would actually watch next? | Yes / No |

If <4/6, revise before delivering. Don't ship a script with weak retention.

## Common Pitfalls

| Problem | Solution |
|---------|----------|
| Hook starts with "Hey guys, welcome back to my channel" | Cut entirely. Open with the result/question/contrarian line. Save channel branding for the mid-roll CTA. |
| Script reads like a blog post (long paragraphs, no visual cues) | Rewrite sentence-by-sentence. Every spoken line ≤18 words. Add a `[VISUAL]` or `[B-ROLL]` marker every 60–90s. |
| No open loop — viewer knows everything by minute 2 | Plant a "I'll show you X later" promise in the first 30 seconds. Pay it off near the climax. |
| Mid-video energy sag — long technical explanation with no visual change | Add a pattern interrupt: cutaway to whiteboard, zoom on screen, B-roll, meme, or sound effect. |
| Title is too generic ("My thoughts on productivity") | Front-load the hook word. Add a specific number or outcome. Test against the "would I click this in my feed?" rule. |
| Thumbnail text is unreadable on mobile | Maximum 3 words. 60+ pt font equivalent. High contrast against background. Test by squinting — if you can't read it, neither can they. |
| End-screen says "smash that like button" with no specific next video | Replace with a specific next-video recommendation tied to the viewer's likely next question. |
| Script length doesn't match audience attention | 8 min for cold audience, 15 min for engaged audience, 25+ min for deep-dive loyal audience. Match the format choice in Step 1. |
| Spoken lines are too long (>20 words) | Break into 2 lines. Add breath. Pause for visual change. |
| No chapter timestamps in description | Always include them — they improve SEO, increase average view duration, and let viewers jump back. |

## Verification Checklist

Before delivering the script to the user:

- [ ] First 8 seconds contain a hook from the 6 proven patterns (no channel intro)
- [ ] At least 2 open loops planted before the first major payoff
- [ ] Pattern interrupt markers (`[VISUAL]`, `[B-ROLL]`, `[ZOOM]`, `[GRAPHIC]`) every 60–90 seconds
- [ ] Every 2–3 minutes contains a tangible payoff (insight, demo, number, or result)
- [ ] Spoken line length averages 12–18 words
- [ ] CTA is specific and tied to a next video, not "like and subscribe"
- [ ] Title is ≤60 characters and front-loads the hook word
- [ ] 2–3 alternate titles provided for A/B testing
- [ ] Thumbnail brief has subject + emotion + 3-word text overlay
- [ ] Description includes chapter timestamps matching the script
- [ ] Retention score ≥4/6 across all 6 signals
